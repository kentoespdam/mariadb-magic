# AGENT.md

## Overview
The **MariaDB Magic** project is driven by an autonomous coding agent that follows the guidelines laid out in `CLAUDE.md`. The agent’s responsibilities include:
- Scanning the codebase with **graphify** to discover functions, logic, and dependencies before any modification.
- Consulting the **Product Requirements Document** (`plan/prd.md`) as the single source of truth for feature specifications.
- Respecting architectural decisions captured in the ADR collection (`docs/adr/`).
- Applying **DRY** principles and using **context7** for up‑to‑date best‑practice libraries.
- Writing code in a **test‑first** manner (TDD) and ensuring race‑detector safety where required.
- Managing work through the **Beads** issue tracker (`bd` commands) and completing the mandatory push workflow.

## Workflow
1. **Code discovery** – Run `graphify query "<search term>"` to locate relevant files, functions, or logic. The agent never falls back to `ls`, `grep`, or `cat` unless graphify reports no results.
2. **Reference check** – Verify the intent against:
   - `CLAUDE.md` – coding conventions and tooling protocols.
   - `plan/prd.md` – the product definition (the "truth").
   - `docs/adr/` – approved architectural decisions.
3. **Library lookup** – Before adding new dependencies, query `context7` for the latest, recommended library versions.
4. **Implementation** – Write concise, DRY Go/Next.js code, accompanied by unit tests.
5. **Issue tracking** – Create or claim work via `bd` commands; close issues once the code is pushed.
6. **Push** – Follow the mandatory push steps (rebase, `bd dolt push`, `git push`, verify status).

## Key References
- **Coding standards & tooling** – see [`CLAUDE.md`](CLAUDE.md).
- **Product requirements** – the authoritative spec is [`plan/prd.md`](plan/prd.md).
- **Architectural decisions** – located in [`docs/adr/`](docs/adr/). These ADR files are the definitive source for design choices.

## Maintaining Truth
Any change to **features**, **behaviour**, or **architecture** must be reflected first in the appropriate source of truth:
- Feature changes → update `plan/prd.md`.
- Design changes → add or modify an ADR in `docs/adr/`.
- Coding conventions → adjust `CLAUDE.md`.
Only after the source of truth is updated should the agent modify implementation files.
