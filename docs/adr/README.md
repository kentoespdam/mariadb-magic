# Architectural Decision Records (ADRs)

This directory contains the **authoritative** architectural decisions for the *MariaDB Magic* project. Each ADR file is the single source of truth for its respective design choice and must be referenced before any implementation change.

## Usage Guidelines for the Coding Agent
1. **Never modify code before consulting an ADR.** If a needed decision is missing, create a new ADR following the existing naming convention (`NNN-description.md`).
2. **Treat the ADRs as immutable unless the product requirements (`plan/prd.md`) change.** When the PRD is updated, amend the relevant ADRs accordingly.
3. **When searching for existing decisions, the agent must use `graphify query "<search term>"`.** Direct filesystem inspection (`ls`, `cat`, etc.) is prohibited.
4. **All new feature work must be traceable to an ADR.** The agent should include the ADR identifier (e.g., `ADR-0012`) in commit messages and issue descriptions.
